## 실전에서의 도구 선택 예시

신규 기능의 효과를 분석한다면 **DiD(Ch18)**가 적절할 수 있다:

```
인과 다이어그램:
  UserMotivation → FeatureAdopted
  UserMotivation → Outcomes
  FeatureAdopted → BetterExperience → Outcomes
  Time → Outcomes (자연적 추세)

문제: UserMotivation은 관찰 불가 → 백도어 완전히 못 닫음

DiD의 접근:
  - 채택 전후의 변화량을 비교
  - 비채택자의 변화량을 "자연적 추세"로 사용
  - (채택자 변화) - (비채택자 변화) = 순수 효과
  - 가정: UserMotivation의 시간 변화가 두 그룹에서 동일
```

**매칭(Ch14)**도 대안이 될 수 있다:
```
  - 채택 유저와 비슷한 특성의 비채택 유저를 매칭
  - 성향 점수(Propensity Score)로 유사성 측정
  - 매칭된 쌍끼리 비교
```

두 방법 모두 **"Stuff를 간접적으로 통제"**하는 전략이다.
