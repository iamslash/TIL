# Chapter 2: 연구 질문 (Research Questions)

> **핵심 질문**: 좋은 연구 질문은 어떻게 만드는가? 데이터 마이닝은 왜 인과 질문에 답하지 못하는가?

---

## 2.1 연구 질문이란?

연구 질문은 두 가지 조건을 만족해야 한다:
1. **답할 수 있어야** 한다 (데이터로 검증 가능)
2. **세상이 어떻게 돌아가는지 이해를 개선**해야 한다

### 나쁜 질문 vs 좋은 질문

| 나쁜 질문 | 이유 | 좋은 질문 |
|----------|------|----------|
| "최고의 007 영화는?" | 주관적, 답할 수 없음 | "어느 시대 007이 티켓 매출이 가장 높았나?" |
| "사회적 이동성의 결정 요인은?" | 범위가 너무 넓음 | "태어난 지역이 사회적 이동성에 영향을 주는가?" |
| "이탈리아 르네상스가 그림에 미친 영향은?" | 너무 방대함 | "원근법을 가장 빨리 도입한 나라의 특징은?" |

### 이론 → 가설 → 검증

```
이론 (Theory)
  "수동적 엔터테인먼트는 호기심을 떨어뜨린다"
    ↓
연구 질문 (Research Question)
  "어릴 때 TV를 많이 보면 성인의 호기심이 줄어드는가?"
    ↓
가설 (Hypothesis)
  "세서미 스트리트를 시청한 아이들의 성인 호기심 점수가 낮을 것이다"
    ↓
데이터 수집 + 검증
```

**좋은 연구 질문의 테스트**: 만약 예상과 반대 결과가 나오면 어떻게 되는가?
- 세서미 스트리트 시청자의 호기심이 **더 높다**는 결과가 나오면?
- 이론을 재검토해야 한다면 → **좋은 질문**
- "그래도 이론은 맞다"고 쉽게 넘어갈 수 있다면 → **나쁜 질문**

## 2.2 왜 질문으로 시작해야 하는가?

### 데이터 마이닝의 한계

데이터 마이닝은 데이터에서 패턴을 찾는 방법이다. 유용할 때도 있지만, 인과
질문에는 답하지 못한다.

#### 한계 1: 상관관계 ≠ 인과관계

```
관찰: 반바지 판매가 많은 날, 아이스크림 매출도 높다
데이터 마이닝: "반바지 → 아이스크림" 관계 발견!
현실: 더운 날씨가 둘 다에 영향 (교란 변수)
```

비디오 게임과 공격성도 마찬가지:
- 데이터에서 상관관계가 나왔다고 "게임이 공격성을 유발한다"고 결론 내릴 수 없다
- 공격적인 성향의 사람이 폭력적 게임을 선택할 수도 있다

#### 한계 2: 추상화 불가능

데이터 마이닝은 "의자"라는 개념을 학습할 수 없다.
- 데이터에는 "평면", "4개의 다리", "나무 재질" 같은 물리적 속성만 있다
- "앉기 위한 물건"이라는 **추상 개념**은 데이터에 없다
- 그래서 4다리 의자와 빈백 의자를 같은 범주로 묶지 못한다

> ChatGPT가 손가락 개수를 틀리는 이유도 같다. 패턴은 있지만 "손 이론"이 없다.

#### 한계 3: 거짓 양성 (False Positives)

100개 변수를 공격성과 비교하면, **순전히 우연으로** 일부는 관련 있어 보인다.
질문 없이 마구 탐색하면 이런 거짓 패턴을 진짜로 착각한다.

#### 데이터 마이닝이 유용한 경우

- **예측**: "이 유저가 이탈할 것인가?" (왜인지는 몰라도 됨)
- **아이디어 생성**: 비아그라는 혈압약 테스트 중 예상치 못한 효과를 발견하여
  아이디어를 얻었다. 하지만 이후 **별도 검증**을 거쳤다.

## 2.3 연구 질문은 어디서 오는가?

### 이론에서 출발

```
이론: "사람은 인센티브에 반응한다"
→ 질문: "성적에 돈을 주면 학생들이 더 열심히 공부할까?"

이론: "식물은 햇빛에서 에너지를 얻는다"
→ 질문: "어두운 방에 둔 식물은 죽을까?"

이론: "CD 매출이 떨어진 이유는 스트리밍 때문이다"
→ 질문: "인터넷이 느린 지역에서 CD가 더 잘 팔릴까?"
```

### 기회에서 출발

- **데이터가 있을 때**: 기존 데이터셋으로 답할 수 있는 질문 찾기
- **현실 사건이 있을 때**: "학교가 성적에 돈을 준다고? 효과가 있을까?"

---

## 2.4 좋은 질문의 체크리스트

| 기준 | 질문 |
|------|------|
| **결과 해석 가능** | 어떤 결과가 나오든 의미 있게 해석할 수 있는가? |
| **실행 가능** | 실제로 데이터를 구할 수 있는가? |
| **규모 적절** | 가용 시간과 자원에 맞는 범위인가? |
| **설계 가능** | 이 질문에 답할 수 있는 연구 방법이 존재하는가? |
| **단순** | 여러 질문을 한꺼번에 묶지 않았는가? |

## Python 예제: 데이터 마이닝의 거짓 양성

```python
import numpy as np
import pandas as pd

np.random.seed(42)
n = 200

# 진실: 어떤 변수도 outcome에 영향을 주지 않는다
outcome = np.random.normal(0, 1, n)

# 100개의 무작위 변수를 만든다
variables = {f'var_{i}': np.random.normal(0, 1, n) for i in range(100)}
df = pd.DataFrame(variables)
df['outcome'] = outcome

# 각 변수와 outcome의 상관관계를 계산
correlations = df.corr()['outcome'].drop('outcome').abs().sort_values(ascending=False)

print("=== 상관관계 상위 5개 (모두 거짓!) ===")
print(correlations.head())
print(f"\n|상관관계| > 0.14 인 변수 수: {(correlations > 0.14).sum()}/100")
print("\n→ 진실은 '아무 관계 없음'이지만, 우연히 관련 있어 보이는 변수가 나옴")
print("→ 이것이 데이터 마이닝으로 인과관계를 주장하면 안 되는 이유")
```

**핵심 교훈**:
- 100개 변수를 테스트하면, 5%의 유의수준에서 약 5개는 **우연히** 유의하게 나온다
- 연구 질문 없이 탐색하면, 이런 거짓 패턴을 "발견"이라고 착각한다
- **질문이 먼저, 데이터가 나중**이어야 하는 이유다

## 소프트웨어 엔지니어를 위한 비유

| 데이터 마이닝 | TDD (테스트 주도 개발) |
|-------------|----------------------|
| 데이터를 먼저 보고 패턴 찾기 | 구현을 먼저 하고 테스트 나중에 |
| "이 패턴이 왜 있는지 나중에 설명" | "이 코드가 왜 되는지 나중에 확인" |
| 거짓 양성 위험 높음 | 회귀 버그 위험 높음 |

인과추론은 **TDD의 연구 버전**이다:
1. 먼저 가설(테스트)을 세운다
2. 그다음 데이터(구현)를 확인한다
3. 가설이 맞는지 검증한다
