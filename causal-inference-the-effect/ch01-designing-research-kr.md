# Chapter 1: 연구 설계하기 (Designing Research)

> **핵심 질문**: 데이터로 질문에 답하려면, 왜 "설계"가 필요한가?

---

## 1.1 나에게 질문이 있다

모든 연구는 **"세상이 어떻게 돌아가는가?"** 라는 궁금증에서 시작한다.

예를 들어:
> "고속도로 차선을 하나 더 추가하면 교통 체증이 줄어들까?"

이것이 **연구 질문(Research Question)** 이다. 좋은 연구 질문은:
- **명확하게 정의**되어 있고
- **답할 수 있고**
- **이해하기 쉬워야** 한다

---

## 1.2 경험적 연구 (Empirical Research)

질문에 답하는 방법은 여러 가지다:
- 문헌 조사 (기존 전문가 의견 검토)
- 철학적 추론 (가정에서 논리적으로 유도)
- **경험적 관찰** (실제 데이터를 수집하여 구조적으로 분석)

이 책은 **정량적 경험적 연구(quantitative empirical research)** 에 집중한다.
즉, 인터뷰보다는 **숫자와 데이터셋**을 다룬다.

### 핵심 문제: 관찰 ≠ 답

고속도로 예제를 다시 보자:
- 기존 2차선 도로와 3차선 도로의 정체를 비교하면 답이 나올까?
- **아니다.** 3차선 도로는 원래 교통량이 많아서 확장된 것일 수 있다.
- 우리에게 필요한 것은 **"만약 이 2차선 도로를 3차선으로 만들었다면?"** 이라는
  반사실(counterfactual)인데, 이건 관찰할 수 없다.

하지만 올바르게 설계하면, **관찰 가능한 데이터에서 인과적 답**을 이끌어낼 수 있다.

---

## 1.3 왜 설계가 필요한가

### 잘못된 설계의 결과

고속도로에 대한 단순 분석:
```
3차선 도로의 평균 정체: 높음
2차선 도로의 평균 정체: 낮음
→ 결론: 차선을 추가하면 정체가 늘어난다 (???)
```

이 결론은 **틀렸다.** 차선이 많은 도로는 원래 교통량이 많아서 확장된 것이다.
이것이 **선택 편향(Selection Bias)** 이다.

### 영양학의 교훈

영양 과학은 설계 실패의 대표 사례다:
- "탄수화물이 좋다" → "탄수화물이 나쁘다"
- "커피가 해롭다" → "커피가 이롭다"
- "버터 대신 마가린" → "마가린 대신 버터"

이런 혼란은 미디어의 과장 때문만이 아니라, **연구 설계 자체가 인과적 질문에
답할 수 없는 구조**였기 때문이다.

저자는 이렇게 비유한다:
> 2 + 2 를 계산하는 대신 다른 계산을 하면 6, 1, -52 같은 답이 나온다.
> 그러면 신문은 "과학자들이 2 + 2 = -52 라고 확인했다"고 보도한다.

## 소프트웨어 엔지니어를 위한 비유

연구 설계는 소프트웨어의 **아키텍처 설계**와 같다.

| 소프트웨어 | 연구 |
|-----------|------|
| 요구사항 정의 | 연구 질문 |
| 아키텍처 설계 | 연구 설계 |
| 구현 | 데이터 수집 + 분석 |
| 테스트 | 결과 검증 (가설 검정) |

코딩을 시작하기 전에 설계를 잘 해야 하듯, 데이터 분석을 시작하기 전에 **연구
설계**를 잘 해야 한다. 설계 없이 데이터를 뒤지면, 고속도로 예제처럼 **"차선을
늘리면 정체가 늘어난다"** 같은 잘못된 결론에 도달한다.

---

## Python 예제: 선택 편향 시뮬레이션

```python
import numpy as np
import pandas as pd

np.random.seed(42)
n = 1000

# 실제 세상: 교통량이 많은 도로에 차선을 추가함
traffic_demand = np.random.normal(100, 30, n)  # 진짜 교통 수요
lanes = np.where(traffic_demand > 110, 3, 2)   # 수요가 높으면 3차선
congestion = traffic_demand - lanes * 20 + np.random.normal(0, 10, n)
# 차선 1개당 정체를 20 줄여줌 (진짜 효과 = -20)

df = pd.DataFrame({
    'traffic_demand': traffic_demand,
    'lanes': lanes,
    'congestion': congestion
})

# 잘못된 분석: 단순 비교
print("=== 단순 비교 (잘못된 분석) ===")
print(df.groupby('lanes')['congestion'].mean())
# 3차선 도로가 더 정체가 심하게 나옴 → 잘못된 결론!

# 올바른 분석: 교통 수요를 통제
import statsmodels.formula.api as smf

model = smf.ols('congestion ~ lanes + traffic_demand', data=df).fit()
print("\n=== 교통 수요 통제 후 (올바른 분석) ===")
print(f"차선의 효과: {model.params['lanes']:.1f}")
# -20에 가까운 값이 나옴 → 차선 추가는 정체를 줄인다!
```

**실행 결과 해석:**
- 단순 비교: 3차선 도로의 정체 평균이 2차선보다 **높게** 나옴 (선택 편향!)
- 교통 수요 통제 후: 차선의 효과는 약 **-20** → 차선 추가는 정체를 **줄인다**
