# Abstract

- Adam(Adaptive Moment Estimation)은 신경망 학습에서 가장 널리 쓰이는 최적화
  알고리즘이다.
- 단순한 경사 하강법(SGD)의 문제점을 모멘텀과 적응적 학습률로 해결한다.
- 논문: [Adam: A Method for Stochastic Optimization](https://arxiv.org/abs/1412.6980)
  (Kingma & Ba, 2014)

# 왜 SGD 만으로는 부족한가?

가장 단순한 최적화 방법인 SGD(Stochastic Gradient Descent):

```python
# 단순 SGD
p.data -= learning_rate * p.grad
```

이 방법의 문제점:

| 문제 | 현상 | 결과 |
|------|------|------|
| **학습률이 모든 파라미터에 동일** | 중요한 파라미터와 덜 중요한 파라미터를 같은 속도로 업데이트 | 비효율적 |
| **진동(oscillation)** | 경사가 가파른 방향으로 왔다갔다 반복 | 학습이 느려짐 |
| **느린 수렴** | 경사가 완만한 방향에서 너무 조금씩 이동 | 시간이 오래 걸림 |

Adam 은 이 세 가지를 모두 해결한다.

# Adam 의 핵심 아이디어 두 가지

## 아이디어 1: 모멘텀 (Momentum) -- 관성

공이 언덕을 내려가는 상황을 상상하라. SGD 는 현재 경사만 보고 발걸음을
내딛는다. Adam 은 **지금까지 이동해온 방향을 기억**하여 같은 방향이면 가속하고,
다른 방향이면 감속한다.

```
m = 기울기의 이동 평균 (1차 모멘트)
  = "지금까지 어느 방향으로 얼마나 이동했는가?"
```

## 아이디어 2: 적응적 학습률 (Adaptive Learning Rate)

기울기가 자주 크게 변하는 파라미터는 **조심스럽게** 업데이트하고,
기울기가 항상 작은 파라미터는 **과감하게** 업데이트한다.

```
v = 기울기 제곱의 이동 평균 (2차 모멘트)
  = "지금까지 기울기가 얼마나 크고 변동성이 컸는가?"
```

# microgpt.py 의 Adam 코드: 라인별 해설

```python
learning_rate, beta1, beta2, eps_adam = 0.01, 0.85, 0.99, 1e-8
m = [0.0] * len(params)  # first moment buffer
v = [0.0] * len(params)  # second moment buffer
```

## 하이퍼파라미터

| 변수 | 값 | 역할 | 비유 |
|------|-----|------|------|
| `learning_rate` | 0.01 | 한 번에 얼마나 크게 이동할지 | 걸음의 최대 크기 |
| `beta1` | 0.85 | 모멘텀 감쇠율. 이전 방향을 85% 유지 | 관성의 강도 |
| `beta2` | 0.99 | 분산 감쇠율. 이전 변동성을 99% 유지 | 지형 기억의 강도 |
| `eps_adam` | 1e-8 | 0 으로 나누기 방지용 아주 작은 수 | 안전 장치 |
| `m` | `[0.0, ...]` | 각 파라미터의 1차 모멘트 버퍼 | 방향 메모리 |
| `v` | `[0.0, ...]` | 각 파라미터의 2차 모멘트 버퍼 | 크기 메모리 |

**왜 `beta1 = 0.85` 인가?** 일반적인 Adam 논문의 권장값은 `beta1 = 0.9` 이다.
microgpt 는 극소 모델에서 빠른 수렴을 위해 `0.85` 를 사용하여 새 기울기에
더 빨리 반응하도록 했다.

---

```python
lr_t = learning_rate * (1 - step / num_steps)
```

## 학습률 스케줄링 (Linear Decay)

```
step=0:    lr_t = 0.01 × (1 - 0/1000)   = 0.01    (최대)
step=500:  lr_t = 0.01 × (1 - 500/1000) = 0.005   (절반)
step=999:  lr_t = 0.01 × (1 - 999/1000) = 0.00001 (거의 0)
```

**왜 학습률을 줄이는가?**

초반에는 파라미터가 최적값에서 멀리 있으므로 큰 걸음으로 빠르게 접근한다.
후반에는 최적값 근처에 있으므로 작은 걸음으로 섬세하게 조정한다. 큰 걸음으로
최적값 근처를 왔다갔다 진동하는 것을 방지한다.

```
loss
 │
 │  \
 │   \        큰 걸음으로          작은 걸음으로
 │    \       빠르게 내려옴  →     조심스럽게 안착
 │     ─────────────────────────────
 └──────────────────────────────── step
      초반 (lr 큼)                후반 (lr 작음)
```

---

```python
m[i] = beta1 * m[i] + (1 - beta1) * p.grad
```

## 1차 모멘트 업데이트 (지수이동평균, EMA)

**수식**: `m_t = β₁ × m_{t-1} + (1 - β₁) × g_t`

여기서 `g_t = p.grad` (현재 스텝의 기울기)

**직관**: 이전 방향을 85% 유지하고, 새 기울기를 15% 반영한다.

**수치 예시** (`beta1=0.85`, 기울기가 계속 `g=1.0` 인 경우):

```
step 0: m = 0.85 × 0.0 + 0.15 × 1.0 = 0.150
step 1: m = 0.85 × 0.150 + 0.15 × 1.0 = 0.278
step 2: m = 0.85 × 0.278 + 0.15 × 1.0 = 0.386
step 3: m = 0.85 × 0.386 + 0.15 × 1.0 = 0.478
...
step ∞: m → 1.0  (기울기와 같아짐 = 완전 수렴)
```

기울기 방향이 갑자기 바뀌는 경우:

```
step 0~4: g=+1.0  →  m 이 +방향으로 쌓임 (≈ +0.56)
step 5:   g=-1.0  →  m = 0.85×0.56 + 0.15×(-1.0) = 0.326  (갑자기 뒤집히지 않음!)
step 6:   g=-1.0  →  m = 0.85×0.326 + 0.15×(-1.0) = 0.127
step 7:   g=-1.0  →  m = 0.85×0.127 + 0.15×(-1.0) = -0.042 (이제야 방향 전환)
```

이것이 **관성(momentum)** 효과이다. 기울기가 갑자기 뒤집혀도 서서히 반응하여
노이즈에 덜 민감하다.

---

```python
v[i] = beta2 * v[i] + (1 - beta2) * p.grad ** 2
```

## 2차 모멘트 업데이트 (기울기 제곱의 지수이동평균)

**수식**: `v_t = β₂ × v_{t-1} + (1 - β₂) × g_t²`

**직관**: 기울기 제곱의 이동 평균이므로, 이 파라미터가 최근에 얼마나 큰 기울기를
경험했는지를 추적한다. 분산(variance)의 추정값이다.

**수치 예시** (`beta2=0.99`):

```
파라미터 A: 기울기가 항상 큼 (g=5.0)
  step 0: v = 0.99×0.0 + 0.01×25 = 0.25
  step 1: v = 0.99×0.25 + 0.01×25 = 0.498
  ...
  step ∞: v → 25.0  (크고 안정적인 기울기)

파라미터 B: 기울기가 항상 작음 (g=0.1)
  step 0: v = 0.99×0.0 + 0.01×0.01 = 0.0001
  step 1: v = 0.99×0.0001 + 0.01×0.01 = 0.000199
  ...
  step ∞: v → 0.01  (작고 안정적인 기울기)
```

나중에 업데이트 크기를 `sqrt(v)` 로 나누므로:
- A: `/ sqrt(25) = /5` → 큰 기울기지만 조금씩 이동 (조심스럽게)
- B: `/ sqrt(0.01) = /0.1` → 작은 기울기지만 크게 이동 (과감하게)

---

```python
m_hat = m[i] / (1 - beta1 ** (step + 1))
v_hat = v[i] / (1 - beta2 ** (step + 1))
```

## 편향 보정 (Bias Correction)

**문제**: `m` 과 `v` 는 모두 0 으로 초기화된다. 초기 스텝에서는 실제보다 훨씬
작은 값을 가진다.

**예시** (`beta1=0.85`, `step=0`, `g=1.0`):

```
m[0] = 0.85×0.0 + 0.15×1.0 = 0.15  ← 실제 기울기(1.0)보다 훨씬 작음!
```

이는 0 으로 시작했기 때문에 생기는 인위적 편향이다.

**편향 보정 수식**: `m̂ = m / (1 - β₁^t)`

```
step=0 (t=1): m_hat = 0.15  / (1 - 0.85¹) = 0.15  / 0.15 = 1.0  ✓ 복원됨!
step=1 (t=2): m_hat = 0.278 / (1 - 0.85²) = 0.278 / 0.278 ≈ 1.0  ✓ 복원됨!
step=9 (t=10): m_hat = m / (1 - 0.85¹⁰) = m / 0.803  (점점 보정이 작아짐)
step=99(t=100): m_hat = m / (1 - 0.85¹⁰⁰) ≈ m / 1.0   (보정 거의 필요없음)
```

`beta2=0.99` 로 더 느린 `v` 는 초기 편향이 더 크다:

```
step=0 (t=1): v_hat = v / (1 - 0.99¹) = v / 0.01   ← 100배로 증폭! (편향 심함)
step=9 (t=10): v_hat = v / (1 - 0.99¹⁰) = v / 0.096 (10배)
step=99(t=100): v_hat = v / (1 - 0.99¹⁰⁰) = v / 0.634
step=699(t=700): v_hat = v / (1 - 0.99⁷⁰⁰) ≈ v / 1.0  (보정 완료)
```

**핵심**: 편향 보정이 없으면 학습 초반에 `v_hat` 이 매우 작아서 업데이트 크기가
폭발적으로 커진다. 편향 보정으로 초기에도 올바른 스케일을 유지한다.

---

```python
p.data -= lr_t * m_hat / (v_hat ** 0.5 + eps_adam)
```

## 파라미터 업데이트

**수식**: `θ_t = θ_{t-1} - α_t × m̂_t / (√v̂_t + ε)`

각 항의 역할:

| 항 | 역할 | 의미 |
|----|------|------|
| `lr_t` | 전체 이동 크기 | 학습률 (선형 감소) |
| `m_hat` | 이동 방향과 크기 | 모멘텀이 적용된 기울기 |
| `v_hat ** 0.5` | 스케일 조정 | `sqrt(기울기 분산)` = 기울기의 RMS |
| `eps_adam` | 안전 장치 | `v_hat ≈ 0` 일 때 0 나누기 방지 |

**`-=` 를 쓰는 이유**: `m_hat` 이 양수면 그 방향으로 가면 loss 가 증가하므로,
반대 방향(빼기)으로 이동해야 loss 가 감소한다.

**전체 수치 예시** (`step=0`, `p.grad=0.5` 가정):

```
lr_t  = 0.01 × (1 - 0/1000) = 0.01

m[i] = 0.85×0.0 + 0.15×0.5 = 0.075
v[i] = 0.99×0.0 + 0.01×0.25 = 0.0025    (0.5² = 0.25)

m_hat = 0.075  / (1 - 0.85¹) = 0.075  / 0.15  = 0.5
v_hat = 0.0025 / (1 - 0.99¹) = 0.0025 / 0.01  = 0.25

업데이트 크기 = lr_t × m_hat / (√v_hat + ε)
              = 0.01 × 0.5 / (√0.25 + 1e-8)
              = 0.01 × 0.5 / (0.5 + ~0)
              = 0.01 × 1.0
              = 0.01

p.data -= 0.01
```

편향 보정이 `m_hat=0.5`, `v_hat=0.25` 로 복원하여, 실제 기울기 방향(0.5)으로
정확히 1 학습률만큼 이동하게 된다.

**적응적 학습률 효과 비교** (같은 `lr_t=0.01`):

```
파라미터 A: 기울기가 크고 일관됨  →  v_hat 큼 (≈25)
  이동 크기 = 0.01 × m_hat / sqrt(25) = 0.01 × m_hat / 5  → 조금 이동

파라미터 B: 기울기가 작고 희박함  →  v_hat 작음 (≈0.01)
  이동 크기 = 0.01 × m_hat / sqrt(0.01) = 0.01 × m_hat / 0.1  → 많이 이동
```

결과적으로 모든 파라미터가 비슷한 스케일로 업데이트된다.

---

```python
p.grad = 0
```

## 기울기 초기화

각 스텝이 끝난 후 모든 파라미터의 기울기를 0 으로 리셋한다.

**왜 필요한가?** microgpt 의 `Value.backward()` 는 `child.grad +=` 로 기울기를
누적한다. 리셋하지 않으면 이전 스텝의 기울기가 현재 스텝의 기울기에 더해진다.

```
스텝 1: loss.backward()  →  p.grad = 0.5  (스텝 1의 기울기)
p.grad = 0 로 리셋 안 하면 →
스텝 2: loss.backward()  →  p.grad = 0.5 + 0.3 = 0.8  (누적 오염!)

리셋 후 →
스텝 2: loss.backward()  →  p.grad = 0.3  (스텝 2만의 정확한 기울기)
```

# Adam 전체 알고리즘 요약

```
초기화:
  m = 0, v = 0  (모멘트 버퍼)

매 스텝 t 마다:
  1. 기울기 계산:   g = ∂loss/∂θ   (backward())
  2. 1차 모멘트:    m = β₁·m + (1-β₁)·g
  3. 2차 모멘트:    v = β₂·v + (1-β₂)·g²
  4. 편향 보정:     m̂ = m/(1-β₁ᵗ),  v̂ = v/(1-β₂ᵗ)
  5. 업데이트:      θ = θ - α·m̂/(√v̂+ε)
  6. 기울기 리셋:   g = 0
```

## SGD vs Adam 비교

| | SGD | Adam |
|--|-----|------|
| 수식 | `θ -= lr × g` | `θ -= lr × m̂ / (√v̂ + ε)` |
| 방향 | 현재 기울기만 | 모멘텀 적용된 기울기 |
| 학습률 | 모든 파라미터 동일 | 파라미터마다 적응적 |
| 진동 | 심함 | 관성으로 감소 |
| 수렴 속도 | 느림 | 빠름 |
| 구현 복잡도 | 매우 단순 | 버퍼 2개, 편향 보정 필요 |

## 각 변수의 역할 한눈에 보기

```
p.grad ──────────────────────────────────→  현재 기울기 (backward 에서 계산됨)
         │                        │
         ↓ ×(1-β₁)               ↓ ² ×(1-β₂)
         │                        │
m ←── β₁·m + ...          v ←── β₂·v + ...
         │                        │
         ↓ ÷(1-β₁ᵗ)              ↓ ÷(1-β₂ᵗ)
         │                        │
       m_hat                    v_hat
         │                        │
         └──────────┬─────────────┘
                    ↓
              m_hat / (√v_hat + ε)   ← 실제 이동 방향과 크기
                    ↓
           p.data -= lr_t × ...      ← 파라미터 업데이트
```
